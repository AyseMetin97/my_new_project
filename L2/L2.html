1 Managing working directory

The Working Directory (WD) is the default folder on your computer where R reads from and saves files. Whenever you import or export data, figures, workspaces, or other files, it is important to know your current working directory.

  • Where am I and where do I want to go?
setwd("my_path")  # Set the working directory
getwd()           # Get the current working directory
  • How to see what objects are in my working environment?
ls()  # Returns a character vector with the names of objects in the current environment
## character(0)
  • How to see what files are in the current working directory?
list.files()  # Lists the files and folders in the working directory
## [1] "data-transformation.pdf" "Images"                 
## [3] "L2.html"                 "L2.Rmd"
dir()         # Another function to list files and folders (similar to list.files())
## [1] "data-transformation.pdf" "Images"                 
## [3] "L2.html"                 "L2.Rmd"
  • How to delete objects in the working environment?
rm(list = ls())  # Delete all objects in the current environment
rm(x)            # Delete the object named x
  • How to save objects or your workspace?
save(x, file = "x.RData")          # Save the object x to a file
save.image(file = "workspace.RData")  # Save the entire workspace

As a general rule:

  • When you open R or the RStudio app, the default WD may be the documents folder (in Windows) or the home directory (e.g., /home/username; in Linux or macOS)

  • This default may be reset at any time from inside RStudio on Tools > Global Options... > General

  • When RStudio is newly open by opening a file (e.g., a .R script file), the WD may be set at that file location (actually my favorite)

  • However, you can set a new WD at any time from within the R code, using the setwd() function.

2 Load data

Before you can start analyzing data in R, you first need to load your dataset. How you do this depends on the format of the file you’re working with—whether it’s a .csv, .xlsx, .txt, .xls, etc.

Step 1: Identify Your File Format

Check the extension of your file:

  • .csv → comma-separated values
  • .txt or .dat → plain text, usually space or tab delimited
  • .xlsx or .xls → Excel spreadsheets
  • .rds → R’s internal single-object format
  • .RData → .rda → R workspace file

Each file type requires a different approach for importing it into R.

Delimited Text Files

data <- read.table("data.txt", header = TRUE, sep = "\t")

  • header = TRUE → First row contains column names

  • sep = "\t" → Tab-separated (use sep = "," for comma)

CSV Files

data <- read.csv("data.csv", header = TRUE)

This is a shortcut for read.table() with sep = ",".

Excel Files (.xlsx/.xls)

# Install package (only the first time)
install.packages("readxl")

# Load package
library(readxl)

# Read Excel file
data <- read_excel("data.xlsx")

Here, you must install the readxl package.

R’s Native Format

# Read .rds file
data <- readRDS("mydata.rds")

# Save an R object in .rds format (optional)
saveRDS(data, file = "mydata.rds")

Use .rds when you want to save and reload a single R object.

RData Files (.RData or .rda)

# Load an .RData or .rda file
load("my_workspace.RData")
  • This function restores all the objects saved in the file into your current environment.
  • It is typically used when you saved multiple objects together using the save() function:
# Save multiple objects
save(object1, object2, file = "my_workspace.RData")

Alternatively (we will see that later), you can use the package readr from the tidyverse:

# Install tidyverse if not already installed
install.packages("tidyverse")

# Load readr package
library(readr)

# Use read_csv from readr
data <- read_csv("data.csv")

This function is faster and prints helpful messages about the data structure.

Step 2: Specify the File Path

If your file is in your working directory, just provide the file name:

read.csv("my_data.csv")

If your file is in another folder, use the relative or absolute path:

read.csv("data/my_data.csv")
read.csv("C:/Users/Angela/Documents/my_data.csv")

Alternatively you can use the toolbar:

Files > Import Datasets >

or replacing the string path with the function file.choose().

Example:

df = read.csv(file.choose())

3 Conditional programming

Conditional statements like if, if…else, and ifelse in R are essential tools for automating tasks and assisting decision making in data science. What follows are a few simple toy examples, but focus on the underlying logic. This will be greatly useful in more advanced applications

The if statement performs an action only if a condition is met:

age = 20
if(age>=18){
  print("Adult")
}
## [1] "Adult"

Sometimes, however, you may need to perform alternative actions

Here is a practical example of the if…else statement

age = 15 
if(age >= 18){ 
  print("Adult") 
} else { 
    print("Minor") 
  }
## [1] "Minor"

In the above example: if age is 18 or older, R will print “Adult”; otherwise (else) it will print “Minor”

When you need to evaluate more than just two alternative conditions, you can use nested conditional statements, that is you combine multiple if…else statements

age = 10 
if (age >= 18) { 
  print("Adult") 
  } else if (age >= 13) { 
  print("Adolescent") 
} else if (age >= 2){ 
    print("Child") 
} else { 
    print("Infant") 
}
## [1] "Child"

All previous examples required to evaluate a single particular condition that might be TRUE or FALSE. However, you often want to apply this type of operation to an entire vector Using if and if…else directly on a vector will NOT work as intended:

age = c(15, 20, 50) 
if(age >= 18){ 
  print("Adult") 
} else { 
    print("Minor") 
  }

To handle such cases you can use the base ifelse() function, that evaluates each element of a vector individually:

age = c(15, 20, 50) 

ifelse(age >= 18, "Adult", "Minor")
## [1] "Minor" "Adult" "Adult"

The ifelse() function can also be nested to manage multiple conditions, such as in the following example:

age = c(2, 28, 15, 1, 4, 67, 42, 14, 7) 

ifelse(age >= 18, "Adult", ifelse(age >= 13, "Adolescent", ifelse(age >= 2, "Child", "Infant")))
## [1] "Child"      "Adult"      "Adolescent" "Infant"     "Child"     
## [6] "Adult"      "Adult"      "Adolescent" "Child"

4 Iterative Programming

Iterative programming allows you to repeat one or a series of actions automatically, for a predetermined number of times or until a condition is met.

Let’s start with understanding the basics of iterative programming with the for loop:

for(i in 1:5){ 
  print(i) 
  }
## [1] 1
## [1] 2
## [1] 3
## [1] 4
## [1] 5
for(i in 1:5){ 
  print(i^2) 
  }
## [1] 1
## [1] 4
## [1] 9
## [1] 16
## [1] 25
set.seed(123) #fix the seed for random number generation
for(i in 1:5){ 
   x = rnorm(n = 30, mean = 0, sd = 1) 
   print(mean(x))
  }
## [1] -0.04710376
## [1] 0.1783383
## [1] 0.02442039
## [1] -0.09388893
## [1] -0.1835804

in the previous example, the for loop displayed the results but didn’t store it. For more effective use, you can combine the for loop with indexing with [] to save each result:

set.seed(0) # set a seed for reproducibility: best practice 
niter = 5 # set the desired number of iterations

# initialize a results vector with NAs: best practice! 
results = rep(NA, niter) # now run the for loop! :-) 
for(i in 1:niter){  
  x = rnorm(n = 30, mean = 0, sd = 1)  
  results[i] = mean(x) 
  } 
results # display results
## [1]  0.021950789 -0.025771530 -0.009581231  0.032123159 -0.294644080

You don’t necessarily have to iterate over a sequence of integers (e.g., i in 1:10000; j in 1:ncol(df)), although this is the most common practice. You could iterate over whatever, for example, directly over the elements of a vector or other data structures

myVect = c("this", "is", "a", "vector", "of", "strings") 

for(x in myVect){ 
  print(toupper(x)) #from lower to upper case
  }
## [1] "THIS"
## [1] "IS"
## [1] "A"
## [1] "VECTOR"
## [1] "OF"
## [1] "STRINGS"

The while loop is another type of iterative structure in R. It may be useful when the precise number of iterations is not predetermined, but depends on a target being reached

# initialize counter
i <- 1

# while loop
while (i <= 5) {
  print(paste("The current value of i is:", i))
  i <- i + 1  # increment
}
## [1] "The current value of i is: 1"
## [1] "The current value of i is: 2"
## [1] "The current value of i is: 3"
## [1] "The current value of i is: 4"
## [1] "The current value of i is: 5"

Traditional loops provide a straghtforward, intuitive way to compute sequences of operations, but the apply family allows you to run faster computations… this may become particularly important when you need to parallelize for computationally intensive tasks.

FYI, other functions within the apply family:

  • tapply(): applies a function to subsets of a vector grouped by a factor (know that the function aggregate() might be more convenient in some cases)
# A numeric vector of ages
age <- c(23, 25, 30, 21, 24, 22)

# A factor indicating gender
gender <- factor(c("F", "M", "M", "F", "F", "M"))

# Compute the mean age for each gender group
tapply(age, gender, mean)
##        F        M 
## 22.66667 25.66667
  • lapply(): applies a function to each element of a list, returning results in a list format, example lapply(my_list, length)
# A list of numeric vectors

# Apply the mean function to each element
lapply(my_list, length)
## $a
## [1] 5
## 
## $b
## [1] 15
## 
## $c
## [1] 25

##  5 15 25

5 Define Custom Functions with function()

Previously, we saw a few cases of custom functions, for example for counting NAs in vectors.

Here is the schema for defining custom functions with input (argument[s]), body, and output (return):

# define new function with some argument(s) as input 
myFunctName = function(arg1 = NA, arg2 = NA, arg3 = TRUE){ 
  # body, series of operations    
  computes several operations     
  variously uses arg1, arg2, arg3     
  get an object named "res" as final result 
  # gives an output 
  return(res) }

Here is a full example:

z_score = function(vect = NA){  
  vectM = mean(vect, na.rm=T)  
  vectSD = sd(vect, na.rm=T)  
  z = (vect - vectM) / vectSD 
  return(z) 
  }

After creating it, a custom function can be used like any other function:

x = c(101, 90, NA, NA, 114, 87, 106, 98, 93) 
z_score(x)
## [1]  0.27162785 -0.89033574          NA          NA  1.64485756 -1.20723491
## [7]  0.79979313 -0.04527131 -0.57343658

6 The tidyverse package

The tidyverse is a collection of R packages that share a common design philosophy and work seamlessly together to make data manipulation, analysis, and visualization more intuitive and efficient. The core idea behind the tidyverse is the concept of “tidy data”, where datasets are organized in a consistent way: each variable forms a column, each observation forms a row, and each type of observational unit forms a table.

This structure simplifies data cleaning, transformation, and analysis because functions expect data in this standard format. The tidyverse includes popular packages such as:

  • ggplot2 : creating graphs

  • dplyr : data manipulation

  • readr : read rectangular data from delimited files, such as comma-separated values (CSV) and tab-separated values (TSV).

  • stringr : manipulate strings

  • …

We will mainly focus on the ggplot2 and dplyr packages.

By adopting the tidyverse, users benefit from:

  • Clear, readable, and consistent syntax,

  • The use of “pipes” (%>%) to chain commands smoothly,

  • Integration among packages to streamline workflows,

  • A focus on human-readable code that’s easier to write and understand.

Overall, the tidyverse encourages a coherent and efficient workflow for data science tasks.

As said before, Tidyverse is based on the pipe operator i.e.,%>%. It connects two function calls by making the value returned by the first call the first argument of the second call:

c(1:3) %>% rep(3) 
## [1] 1 2 3 1 2 3 1 2 3

returns the same output as

rep(c(1:3), 3)
## [1] 1 2 3 1 2 3 1 2 3

We can use the pipe more than one times:

c(1:3) %>% rep(3) %>% length()
## [1] 9

which in basic R is:

length(rep(c(1:3), 3))
## [1] 9

By default the value of the left-hand call is piped into the right-hand call as the first argument. You can make it some other argument by referring to it as the dot ., for example:

3 %>% rep(c(1:3), .) 
## [1] 1 2 3 1 2 3 1 2 3

6.1 dplyr

Inside the tidyverse function we have the useful dplyr package. We will see only the main functions of this package, but you can use the basic R command proposed before instead of these ones.

  • filter(): Subset rows using column values

  • select(): Subset columns using their names and types

  • mutate(): Create, modify, and delete columns

  • group_by(): Group by one or more variables

  • summarize(): Summarize each group to fewer rows
